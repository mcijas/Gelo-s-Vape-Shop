Database Setup Instructions

Your dashboard is not showing any data because there's an issue with the database connection. Follow these steps to fix the problem:

1 Start XAMPP Control Panel

First, you need to make sure XAMPP is running:

  1. Find and open the XAMPP Control Panel (usually located at C:\xampp\xampp-control.exe)
  2. Start the Apache and MySQL services by clicking the "Start" buttons next to them
  3. Make sure both services show a green status
Note: If you don't have XAMPP installed, you'll need to download and install it first.
2 Create the Database

Once XAMPP is running, you need to create the database:

  1. Open your web browser and go to http://localhost/phpmyadmin
  2. Click on "New" in the left sidebar
  3. Enter gelo_pos as the database name
  4. Select utf8mb4_unicode_ci as the collation
  5. Click "Create"
3 Initialize the Database

After creating the database, you need to initialize it with tables and sample data:

  1. Open your web browser and go to http://localhost/12/api/init_db.php
  2. This script will create all necessary tables and add sample data
  3. You should see a success message confirming the database has been initialized
4 Verify Database Connection

To make sure your database connection is working properly:

  1. Check that the database connection settings in api/db.php match your XAMPP configuration:
    2. $host = '127.0.0.1';
$db   = 'gelo_pos';
$user = 'root';
$pass = '';
  2. These are the default settings for XAMPP. If you've changed your MySQL password, update the $pass value.

Common Issues and Solutions

MySQL Service Not Running

If you see an error about MySQL connection failing:

  • Check if the MySQL service is running in XAMPP Control Panel
  • If it won't start, check the XAMPP error logs
  • Try restarting your computer and then starting XAMPP again
Wrong Database Name

If you're using a different database name:

  • Edit api/db.php and change the $db variable to match your database name
  • Make sure the database exists in phpMyAdmin
Port Conflicts

If XAMPP services won't start due to port conflicts:

  • Open XAMPP Control Panel
  • Click on "Config" for the service that won't start
  • Change the port numbers to unused ports
  • Restart the services
